V1 editor surface

Scene Canvas

Main workspace for pan, zoom, place nodes, move nodes between areas, draw relations, and inspect the world visually.

Inspector Panel

Authoritative editor for fields, memberships, relation properties, validation errors, and quick action confirmation.

Structure Navigator

Secondary outline for focus, search, collapse, and quick jumping across large graphs. Support view, not the primary scene model.

Template Picker

Load a schema-defined starter world or starter subgraph, then continue editing manually.

Relation Tools

Create and edit typed edges, including visual differentiation for structure edges versus general relation edges.

Status Bar

Show dirty state, save state, sync state, and validation counts so the host product can surface reliability clearly.

Testing strategy

Quality gates for a reusable module
Core unit tests
command semantics, schema validation, structure-edge rules, template expansion, undo/redo, snapshot recovery
Adapter contract tests
local adapter save/load behavior, sync adapter event contract, pending change recovery, revision mismatch handling
React integration tests
provider lifecycle, selector updates, command dispatch from components, host subscription behavior
Editor interaction tests
drag node, drag into container, confirm candidate action, cancel candidate action, create relation, inspector edits
Performance checks
browse and edit graphs in the target B range: hundreds to low thousands of nodes, with relation-heavy scenes

Explicit V1 non-goals

1

No natural-language graph generation yet

The module will expose command APIs that an LLM can call later, but V1 does not include NL parsing or agent orchestration.

2

No real-time multi-user collaboration

Remote sync boundaries are designed in, but live collaborative editing is deferred.

3

No hardcoded writing domain model

Writing presets exist as schema and template examples, not as product-level assumptions baked into the engine.

4

No backend service requirement

The module must work in local-only mode inside a host product.

Success criteria for V1

When this milestone is done
A host React app can embed the editor, load a schema + template, create and edit a graph visually, persist it locally, restore it correctly, and prepare for later remote sync without changing the core model.
