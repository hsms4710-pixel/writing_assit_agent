Why this split

Core stays reusable

No React dependency. The graph model, commands, persistence ports, and sync rules can be reused by other apps or wrapped for other frameworks later.

React remains thin

The React package only connects host apps and UI to the core store. That keeps state logic out of components and avoids lock-in to one editor shell.

Editor is optional

A host can embed the full editor, just the canvas, just the inspector, or only consume the core and build a custom UI.

Design Decisions Locked In

1

Graph-first, not tree-first

The tree view is a projection based on structure edges. The source of truth is still a generic property graph.

2

Local-first by default

All editing works locally. Remote sync is an adapter, not a requirement for using the module.

3

Composable UI surface

Canvas, tree, inspector, and template workflows are separate exportable widgets, not one inseparable page.
